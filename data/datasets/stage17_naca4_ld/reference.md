# Dataset — stage17-naca4-ld (platform-own CFD)

- **Origin:** `platform-validated` — the platform's OWN ground-truth CFD (CONSTITUTION
  Invariant 11). NOT a foreign corpus; every row is a solve this platform generated and
  validated. This dataset may certify a `validated` surrogate.
- **Generated by:** `scripts/stage17_corpus.py` (Stage 17, ADR-032), 2026-07-24.
- **Case:** `aero.optimize.turbulent_airfoil.ShapedTurbulentAirfoil` — NACA-4 camber
  (max_camber `m`, camber_position `p`), k-omega SST, Re = 5e5, AoA = 4°, base grid,
  `end_time` 3000 (SIMPLE), OpenFOAM-ESI v2412 (`openfoam-esi.sif`), on aero-dev.
- **Sampling:** 40-point seeded Latin hypercube (seed 170) over the Stage-15 design space
  (m ∈ [0, 0.08], p ∈ [0.2, 0.6]) + two anchors: the m = 0 baseline and the Stage-15
  optimum (m ≈ 0.0727, p ≈ 0.2045). 42 solves, 0 failures.
- **Objective:** lift-to-drag ratio (`L/D`). Range [21.674, 46.337] (the baseline and
  optimum anchors reproduce the Stage-15 recorded values to four decimals).
- **Schema:** `aero.optimize.corpus.Stage17Corpus` — each `CorpusRow` carries the physical
  and unit-cube design, the L/D, the four-fold provenance tuple, the MLflow run id, and the
  solve wall time. Failed solves (none here) are recorded as evidence rows, never dropped.
- **Provenance:** every row carries a clean-tree four-tuple (git_sha, dvc_input_hash,
  container_sif_sha256, config_hash); MLflow experiment `aero-provenance`, stage 17.
- **Use:** the training corpus for the Stage-17 surrogate (`gp_bootstrap` ensemble). The
  Invariant-9 data gate targets `dataset_hash(repo_root, "data/datasets/stage17_naca4_ld")`.
  `corpus_v2.json` (added in Phase 5) appends the surrogate arms' infill evaluations — the
  own-data flywheel growing.
