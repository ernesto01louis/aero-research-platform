## 2. Decisions made

- **C-grid rebuild (operator decision).** The Stage-03 four-block O-grid is
  retired; the airfoil mesh is now an eight-block multi-block C-grid —
  rectangular far field at 100 chords, an explicit wake cut downstream of the
  TE, the surface split at mid-chord (so upper/lower `polyLine` edges never
  collide on a shared edge key). Front/wake blocks use `edgeGrading` so the
  boundary-layer clustering applies only on the airfoil-side η edge (a uniform
  `simpleGrading` there gave a 2e7 cell aspect ratio). ADR-005 supersedes
  ADR-003's O-grid decision.
- **Tolerances (operator decision):** the prompt's bands — Cd 3 %, Cf 5 %,
  Cp 3 % — accepted as-is.
- **CI split (operator decision):** `vv-required` runs single-grid per case on
  PRs; the GCI mesh sweep is `mesh_sweep`-marked and runs nightly in
  `vv-smoke` only.
- **V&V cases are Python-defined, not Hydra `conf/case/` files.** The case
  classes own the spec because `MeshSweep` needs to scale resolution
  programmatically (`refined(ratio)`); a parallel Hydra file would duplicate
  and drift. `config_hash` for a V&V run is computed from `spec.model_dump()`.
  Deviation from the plan — no `conf/case/{flat_plate_te,...}.yaml`, no
  `_tmr_spec_from_cfg`.
- **TMR geometries — a discriminated union, not an extended `CaseSpec`.**
  `FlatPlateSpec` / `Bump2DSpec` keyed on a `geometry` discriminator
  (`TMRCaseSpec`); the airfoil `CaseSpec` is unchanged but for the C-grid
  fields. Keeps FAIL-LOUD (no optional-and-ignored fields).
- **`aero[vv]` extra** carries `scipy` (pointwise spline interpolation),
  imported lazily — `import aero.vv` stays core-clean.
- **`MeshSweep` uses `BenchmarkRunner.measure_scalar`, not `.run`** — a GCI is
  verification (solution vs. itself), so it needs no validation reference data.
- **Reference data committed in-tree, not DVC-tracked** (deviation — §3).
- **Wall treatment** switched to `nutLowReWallFunction` (resolved-wall) and the
  freestream eddy-viscosity ratio to the TMR value 0.009 — see §6, both turned
  out numerically near-neutral here but are the correct settings.

## 6. Gotchas discovered

- **The NACA 0012 +21 % is pressure drag, not friction.** A `forces` function
  object split it: Cd_friction 0.0067 (correct, ~+2 %), Cd_pressure 0.0031
  (~2× the expected ~0.0015). The skin-friction physics — BL, turbulence
  model, y+, wall treatment — is right; the excess is spurious *pressure*
  drag, traced to the trailing-edge region (the C-grid still has ~28 severely
  non-orthogonal faces, max ~89°, at the TE block corner where four blocks and
  the wake cut meet). This is why the wall-treatment and freestream-turbulence
  changes had ~zero effect (the friction was never the problem).
- **At y+ < 1, `nutkWallFunction` and `nutLowReWallFunction` are numerically
  identical** — the high-Re log-law `nutkWallFunction` clamps to 0 below the
  buffer layer, exactly what the low-Re function returns. Two BC variants gave
  Cd bit-identical to 11 figures. (Kept `nutLowReWallFunction` — correct for a
  resolved mesh.)
- **The flat-plate Cf is compared against an analytic correlation, not CFD
  data.** Turbulent flat-plate Cf correlations span ~10 % among themselves, so
  a 5 % tolerance against the White correlation is tighter than the
  correlation's own spread. The measured Cf is ~7–15 % off. A near-LE
  comparison window (`_LE_WINDOW = 0.1`) drops the x→0 Cf singularity, whose
  spline overshoot otherwise inflated the error.
- **The 2D bump solve stalls.** Its long channel (51 chords) with a
  fully-resolved lower wall gives inlet cells of aspect ratio ~1e5; GAMG hits
  its 1000-iteration cap without converging the pressure equation (p residual
  plateaus ~6e-3). The bump needs a PCG/DIC pressure solver or a coarser
  near-wall inlet before its GCI sweep will converge.
- **blockMesh `edgeGrading` order** is 12 ratios: 4 x1, 4 x2, 4 x3; the x2
  (eta) tuple is `(v0→v3, v1→v2, v5→v6, v4→v7)` — the airfoil-side η edge is
  `v1→v2` for the front blocks but `v0→v3` for the wake blocks.
- **A C-grid airfoil surface must be split at mid-chord** — a single edge
  `(LE, TE)` cannot carry two different `polyLine` curves (upper vs. lower).
- **`apptainer exec --bind`** must use the *in-LXC* path (`/mnt/aero/runs/...`),
  not the host NFS path (`/mnt/aero-nfs/runs/...`).
