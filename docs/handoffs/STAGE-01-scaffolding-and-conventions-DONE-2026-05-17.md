## 2. Decisions made

- **License: GPL-3.0** (per ADR-001). OpenFOAM-aligned. AGPL-3.0 considered and rejected
  because the SaaS-loophole closure buys little for a research platform with no SaaS plans;
  AGPL also creates friction with some downstream OSS projects.
- **Branch protection: pragmatic** with `enforce_admins: false`. Solo-developer escape valve;
  procedural review via the 24-hour cooling-off rule in `CONTRIBUTING.md`. Rejected the
  strict (no admin bypass) variant because it would deadlock the sole developer.
- **mypy strictness: strict on `aero/` only**, lenient elsewhere via
  `[[tool.mypy.overrides]]` in `pyproject.toml`. Tests `ignore_errors=true`. Per-module
  relaxations of the aero-strict rule require an ADR.
- **Commit convention: Conventional Commits scoped to `stage-NN`** (e.g.
  `feat(stage-01): ...`). Enforced by `commit-lint` workflow on every commit + PR title.
- **Pre-commit `subject-case` rule disabled.** Proper nouns (OpenFOAM, NASA, MLflow,
  PhysicsNeMo, Claude Code, RunPod, etc.) appear constantly; enforcing lowercase subjects
  is impractical. Body and footer line-length rules also disabled to allow long URLs and
  Co-Authored-By lines.
- **GitHub-hosted runners only for now.** Self-hosted runner registration deferred to
  Stage 03+. `vv-smoke.yml` is a placeholder echoing "not yet wired".
- **`aero-orchestrator-consumer` repo rename**: the pre-existing `aero-research-platform`
  on GitHub was the orchestrator-consumer project (Apache-2.0, `ai_orchestrator_client`
  dep). Renamed to free the name for this standalone build. The prior project's URLs
  auto-redirect; the local clone sits quarantined at
  `/root/projects/aero-research-platform.PRIOR-REMOTE-CLONE-do-not-touch/`.
- **Operator preference: auto-mode** captured in memory at
  `~/.claude/projects/-root/memory/feedback_auto_mode_aero_platform.md`. Skip explicit
  `approved` prompts on the routine destructive ops listed in the bundle's
  PROPOSE-FIRST sections; still announce major actions; still flag truly high-blast-radius

## 6. Gotchas discovered

- **commitlint `subject-case` default** rejects mixed-case proper nouns. Disabled in the
  workflow's inline config. **Document this in `CONTRIBUTING.md` future revisions** — new
  contributors might expect strict lowercase per common Conventional Commits practice.
- **`pre-commit run --all-files` only sees TRACKED files.** Counter-intuitive when
  scaffolding a brand-new repo — first run reports "no files to check". Workaround: `git
  add -A` first, then run pre-commit. Documented implicitly in the Stage-01 workflow.
- **Pre-commit pins must match actual git tags.** `validate-pyproject` had no `v0.24.2`
  tag (the upstream version 0.24.2 wasn't released; latest in the 0.24 line is `v0.24.1`).
  Always verify with `gh api repos/<owner>/<repo>/tags` when pinning.
- **gh repo create with `--add-readme` / `--license` is NOT what we want for an empty
  repo for Stage-01-style bootstrap.** Confirmed: pure empty repo is required so the
  scaffolding commits hold the canonical LICENSE.
- **Empty initial commit on `main`** is the only way to satisfy "no direct push to main
  except bootstrapping ref" while still allowing the first feature-branch PR. Documented
  here.
- **The `Stop` hook bootstrap window** in `scripts/check_handoff_exists.sh` is set to
  24 hours — if Stage 01 takes longer than 24h between `.aero-stage` creation and the
  handoff existing, Stop will block. Operator can either (a) bump the window in the
  script, (b) update the `.aero-stage` mtime via `touch`, or (c) commit a `status:partial`
  handoff explaining what's blocked.
- **The `aero-orchestrator-consumer` quarantine**: the renamed repo's old URLs auto-
  redirect via GitHub. The local clone at
  `/root/projects/aero-research-platform.PRIOR-REMOTE-CLONE-do-not-touch/` is intact;
  its `git remote` will follow the redirect. **Do not modify.**
